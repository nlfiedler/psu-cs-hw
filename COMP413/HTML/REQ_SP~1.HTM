Requirements Specification

  1. Menu Interface:
    The menu interface shall consist of a menu bar at the top of the screen. Keywords (e.g. File, Vertex, Edge, Search, MST) will appear on the bar. The keyword will be accessed via a mouse or keyboard. Each keyword in turn will have a pull-down menu with actions pertaining to the keyword.
  2. File Operations:
    From main menu keyword "File" a new pull down menu will have these keywords: New/Open, Close, Save, Save As, Backup, Other, Print, About, and Exit. Access will be via the mouse or keyboard.
    1. New/Open - Prompt for file type. File type selection will be via mouse. Once file type has been selected the user will be promoted for file path and name via keyboard input.
      1. Vertex List - for complete graphs
      2. Adjacency Matrix
      3. Adjacency List
      4. Edge List
    2. Save - The file being edited will be saved to the disk and overwrite the file on the disk. Command will be initiated via the mouse.
    3. Save As - From Save As a new pull down menu will have these file types: File type selection will be via mouse. Once file type has been selected the user will be prompted for file path and name via keyboard input. The graph will be saved to disk as an alternative format data file.
      1. Vertex List - for complete graphs
      2. Adjacency Matrix
      3. Adjacency List
      4. Edge List
      File type selection will be via mouse. Once file type has been selected the user will be prompted for file path and name via keyboard input. The graph will be saved to disk as an alternative format data file.
    4. Exit - Quit the program (will save open files fisrt).
  3. Vertex operations.
    1. Function: Random Generation
      Description: Generate a random (x,y) point using the system random number generator.
      Inputs: prompt for n, the number of vertices to be randomly generated. Range of n (1...50).
      Source: User input from the keyboard
      Output: A vertex with center at the (x,y) point.
      Destination: The graph and its' file.
      Requires: N/A
      Pre-condition: No conflicting vertex in the same area.
      Post-condition: Graph will have n additional vertices.
      Side-effects: None
      Rationale: Allows a random graph to be set up.
    2. Function: Add
      Description: Adds a vertex to the current graph. The user selects its position.
      Inputs: Vertex position, Vertex identifier.
      Source: Vertex position is selected by user mouse click. Vertex identifier by keyboard input.
      Output: Vertex is added to the graph.
      Destination: Vertex is added to the screen.
      Requires: N/A
      Pre-condition: No conflicting vertex in the same area.
      Post-condition: Graph will have an additional vertex.
      Side-effects: None
      Rationale: Allows the user to create and modify a graph as desired.
    3. Function: Remove a vertex from the current graph
      Description: Remove the selected vertex from the current graph.
      Inputs: Vertex position or vertex identifier.
      Source: Vertex position is selected by user mouse click. Vertex identifier by keyboard input.
      Output: Vertex is removed from the graph.
      Destination: Vertex is removed from the screen.
      Requires: N/A
      Pre-condition: The graph with one less vertex.
      Post-condition: The edges to/from the deleted vertex must be removed via edge operations.
      Side-effects: None
      Rationale: Allows the user to modify a graph as desired.
  4. Edge operations.
    1. Function: Random Generation
      Description: The system will generate n edges between randomly selected vertices.
      Inputs: Prompt for the number of vertices to create, n. Limit the vertex's (x,y) coordinates by the point size of the document (now set at 1000x1000 points).
      Source: Number will be obtained from keyboard input.
      Output: New edges are added to graph.
      Destination: New edges are drawn to screen.
      Requires: N/A
      Pre-condition: If there is an existing edge between vertices no new edge will be added.
      Post-condition: A pair of vertices with a new edge.
      Side-effects: None
      Rationale: Allows a random graph to be set up.
    2. Function: Add Edge
      Description: Adds an edge to the current graph. The user selects its position by selecting the two vertices the edge will connect.
      Inputs: The two vertex identifier.
      Source: Edge position is selected by user mouse click or keyboard input.
      Output: Edge is added to the graph.
      Destination: Vertex is added to the screen.
      Requires: An additional menu that will let the user choose directed or undirected edge.
      Pre-condition: If directed edge is selected the first input will be the "from ", the next input will be the "to".
      Post-condition: A pair of vertices with a new edge.
      Side-effects: None
      Rationale: Allows the user to create and modify a graph as desired.
    3. Function: Remove an edge
      Description: Remove an edge from the graph.
      Inputs: The two vertices from which the edge is to be removed.
      Source: The user will select the edge using the mouse or keyboard.
      Output: The graph will have one less edge.
      Destination: The screen will reflect the removal of the edge.
      Requires: N/A
      Pre-condition: N/A
      Post-condition: The graph with one less edge
      Side-effects: N/A
      Rationale: Allows the user to modify a graph as desired.
  5. Search operations.
    1. Function: DFS
      Description: Perform a Depth First Search on a graph to demonstrate its application.
      Inputs: The user will be prompted to specify the root vertex and goal vertex.
      Source: Current graph and user input.
      Output: An edge list.
      Destination: Screen will be updated.
      Requires: N/A
      Pre-condition: None
      Post-condition: None
      Side-effects: None
      Rationale: To demonstrate its application in graph theory.
    2. Function: BFS
      Description: Perform a Breath First Search on a graph graph to demonstrate its application.
      Inputs: The user will be prompted to specify the root vertex and goal vertex.
      Source: Current graph and user input.
      Output: An edge list.
      Destination: Screen will be updated.
      Requires: N/A
      Pre-condition: None
      Post-condition: None
      Side-effects: None
      Rationale: To demonstrate its application in graph theory.
    3. Function: Both
      Description: Perfom both a Depth-First and Breath-First searches on a graph to demonstrate their application and difference.
      Inputs: The user will be prompted to specify the root vertex and goal vertex.
      Source: Current graph and user input.
      Output: An edge list.
      Destination: Screen will be updated.
      Requires: N/A
      Pre-condition: None
      Post-condition: None
      Side-effects: None
      Rationale: To compare both methods at the same time.
  6. MST operations.
    1. Function: Kruskal's MST
      Description: Performs Kruskal's MST on the graph one edge at a time.
      Inputs: None
      Source: Current graph.
      Output: An edge list.
      Destination: Screen will be updated to show MST.
      Requires: N/A
      Pre-condition: N/A
      Post-condition: N/A
      Side-effects: None
      Rationale: To demonstrate the application of the algorithm.
    2. Function: Prim's MST
      Description:
      Inputs: Prompt for starting vertex.
      Source: Current graph and user input.
      Output: An edge list.
      Destination: Screen will be updated to show MST.
      Requires: N/A
      Pre-condition: N/A
      Post-condition: N/A
      Side-effects: None
      Rationale: To demonstrate the application of the algorithm.
    3. Function: Both
      Description: Allow for both methods to run simultaneously, each one doing one interation at a time.
      Inputs: None, the starting vertex for Prim's MST will be the first vertex selected by Kruskal's method.
      Source: Current Graph.
      Output: An edge list.
      Destination: Screen will be updated.
      Requires: N/A
      Pre-condition: N/A
      Post-condition: N/A
      Side-effects: None
      Rationale: To compare both methods at the same time.
  7. Other operations.
    1. Function: Shortest Path
      Description: Selects the shortest path between a starting vertex and a goal vertex.
      Inputs: The user will be prompted to specify the root vertex and goal vertex.
      Source: Current graph and user input.
      Output: An edge list.
      Destination: The screen will be updated.
      Requires: N/A
      Pre-condition: N/A
      Post-condition: N/A
      Side-effects: None
      Rationale: To show it's application in graph theory.
  8. Component selection standards.
    1. Specify a vertex by using a mouse.
    2. Specify an edge by choosing two vertices and both mouse buttons. To specify a directed edge click and hold the left mouse button on first vertex and move the mouse to the other vertex and release the mouse button. To specify an undirected edge click and release the left mouse button on first vertex and move the mouse to the other vertex and click and release the mouse button again.
  9. During any interative operation, if the user enters 'a', then switch to an automatic mode in which the operation runs to completion.
