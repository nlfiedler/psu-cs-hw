Non-functional Requirements Definition

  1. User environment will be GEOS because the user interface will be far superior to any DOS interface we've encountered.
  2. The total number of vertices will be limited to 50 to keep the number of vertices on screen reasonable.
  3. Provide Graph visual representation standards.
    1. The numbering standard for the vertices will be specified by their 2D coordinates.
    2. Directed edges will be specified by arrows pointing to the receiving vertex. Undirected edges will have no arrows.
  4. Provide for a means to zoom in and out.
  5. User Input:
    The main method to modify a graph will be by mouse input. Mouse input will determine where the vertex or edge will be located on the screen. All tasks that involve using the mouse for selection will have an alternate keyboard method that will involve specifying the (X,Y) 2D coordinates of the vertex involved.
    Rationale: The user must be able to modify the graph.
  6. Component selection standards.
    1. To select a vertex, use a mouse and click near the vertex to be selected.
    2. To select an edge, by choosing the two connected vertices.
      1. To select a directed edge, click and hold the left mouse button on first vertex and move the mouse to the other vertex and release the mouse button.
      2. To select an undirected edge, click and release the left mouse button on first vertex and move the mouse to the other vertex and click and release the right mouse button again.
  7. During any interative operation provide a keyboard command to switch from interative to automatic.
